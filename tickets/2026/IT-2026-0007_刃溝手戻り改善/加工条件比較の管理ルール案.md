## 加工条件比較の管理ルール（案）

### 目的
- 加工条件を変えた時に、あとで同じ見方で比較できるようにする
- シミュレーション結果と実機結果をひも付けて、採用判断を残せるようにする
- 条件改善を感覚ではなく、記録ベースで判断できるようにする

### 管理の基本単位
- **1条件変更につき1記録**
- 管理の起点は **シミュレーションNo または ファイル名** とする

### 必須記録項目
- 記録日
- 記入者
- 対象設備
- 対象工程
- シミュレーションNo または ファイル名
- 条件差（何を変えたか）
- 確認観点
- 実機結果
- 判定
- 判定者
- 備考

### 記入ルール
- **条件差** は1行で書く  
  例：砥石スリーブ3枚つけ、粗砥石2枚使い
- **確認観点** は増やしすぎず、必要最小限にする  
  例：加工可否、干渉有無、安全性
- **実機結果** は事実だけを書く  
  例：加工可、向心度良好、要確認
- **判定** は次の3つから選ぶ  
  - 採用
  - 不採用
  - 継続確認
- **備考** には、現場判断や追加確認事項だけを書く

### 運用ルール
- 同じ条件を再実施した場合も、別日なら別記録で残す
- 採用判断した条件は、後で判定基準1枚または標準作業へ反映する
- 不採用条件も消さずに残す
- ファイル名だけで管理が不安定な場合は、シミュレーションNoを併記する

### シミュNo採番ルール
- 形式：`SIM-設備名-YYYYMMDD-連番`
- 例：`SIM-MG30-2-20260325-01`

### ルール
- `SIM` はシミュレーション記録を表す固定文字列
- 設備名は対象設備を入れる
- 日付はシミュレーション実施日を8桁で入れる
- 連番は当日内で `01` から採番する
- 同日で複数条件を回した場合は、`02`, `03` と増やす
- 再実施でも別記録とし、新しい番号を採番する
- 条件内容はシミュNoに埋め込まず、管理表の「条件差」欄に記入する

### 管理表のひな形
| 記録日 | 記入者 | 対象設備 | 対象工程 | シミュNo / ファイル名 | 条件差 | 確認観点 | 実機結果 | 判定 | 判定者 | 備考 |
|---|---|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |  |  |  |

### 今回の仮ルール
- 3枚つけは、**2Wトライアル完遂ではなく、シミュ判断移行について現場OKなら完了扱い**とする
- 干渉確認は、現場合意が取れれば **3Dシミュレーション結果を主記録** とする
- 実機側では、加工可否と必要最小限の確認結果のみ残す
